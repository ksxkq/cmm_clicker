# Editor MVP v1（动作列表 + 节点基础属性）

更新时间：2026-03-02

## 1. 目标

1. 提供可运行的编辑器数据层，不依赖流程图视图也能先完成基础编辑。
2. 先打通核心编辑链路：选择 flow -> 选择节点 -> 修改节点 -> 校验反馈 -> 运行验证。
3. 提前落地 `undo/redo`，为后续复杂交互（连线、拖拽）打基础。

## 2. 当前实现范围

## 2.1 Store 层

文件：

1. `app/src/main/java/com/ksxkq/cmm_clicker/feature/editor/TaskGraphEditorStore.kt`

能力：

1. `TaskGraphEditorState`：输出当前 bundle、选中 flow/node、undo/redo 状态、图校验结果。
2. 选中能力：`selectFlow` / `selectNode`。
3. 节点操作：新增动作节点、删除当前节点、上下移动节点。
4. 节点属性编辑：`kind/actionType/pluginId/enabled/active/params`。
5. 历史能力：`undo` / `redo`。
6. `reset`：重置到示例图数据。
7. Flow 管理：`addFlow/renameSelectedFlow/deleteSelectedFlow/setSelectedFlowEntryNode`（含删除保护与入口节点切换）；删除接口返回结构化结果，便于 UI 呈现精确失败原因。

## 2.2 UI 层

文件：

1. `app/src/main/java/com/ksxkq/cmm_clicker/ui/MainActivity.kt`

能力：

1. 页面模式切换：`任务` / `控制台`（`编辑器` 保留为开发调试页）。
2. 编辑器操作面板：undo/redo、新增、删除、移动、重置。
3. Flow 列表选择与节点列表选择。
4. 节点属性编辑面板（kind/actionType/pluginId/flags/params）。
5. Jump 类节点目标选择器（flow/node 点选，不再仅靠手输参数）。
6. Branch 节点支持 `variableKey`、`operator`、`expectedValue` 与 TRUE/FALSE 目标边点选。
7. 节点列表显示 jump 目标摘要（`-> targetFlow/targetNode`）。
8. 图校验面板（显示校验问题列表）。
9. 流程图预览（第一版）：节点/边/jump 同 flow 虚线连线、点击节点选中、相关连线高亮。
10. 参数编辑 schema 化：按节点类型渲染字段，`operator` 走枚举按钮，数值字段走数字键盘。
11. 连线文本预览面板（边与 jump 引用文本）。
12. 运行按钮复用当前编辑后的 bundle（不是固定样例）。
13. 任务页新增动作浮窗链路：简版动作列表 + 独立动作编辑浮窗。
14. 编辑器已接入任务仓库：选中任务加载、自动保存与手动保存。
15. kind 切换时参数会按目标类型重置，避免 `jump` 仍显示 `click` 参数。
16. 动作类型按钮只展示当前已实现动作：`click/dupClick/swipe/record/closeCurrentUI`。
17. `jump` 统一通过 `kind=JUMP` 进行配置，目标由 `targetFlowId/targetNodeId` 点选。
18. 全局浮窗编辑界面使用统一 Dialog Scaffold（Compose 实现）与 Material3 按钮体系，支持头部菜单 action 与底部操作按钮分区。
19. 动作列表支持同 flow 的 jump 连线，可在列表态直接观察跳转关系。
20. 浮窗编辑器交互动画升级：背景渐暗 + 底部上滑进场，作为后续编辑/弹窗交互的统一动画基线。
21. 浮窗支持点击遮罩关闭，并带退出动画（背景淡出 + 面板下滑），统一弹窗开合体验。
22. 参数 schema 升级到 v1.1：字段支持 `required/default/min/max/helperText`，主页面与浮窗编辑器统一展示校验错误提示、辅助文案、数值键盘。
23. 动作类型切换会清理旧类型遗留参数并补齐新类型默认参数，避免 `click` 参数残留到 `swipe` 等类型。
24. 全局浮窗中“动作列表 -> 编辑详情”改为堆叠弹窗转场：底层列表保留并缩放/变暗，上层详情弹窗上滑叠加，返回时反向过渡，强化“进入次级弹窗”感受。
25. 全局浮窗参数输入接入键盘避让：容器 `imePadding` + 输入框聚焦自动 `bringIntoView`，降低键盘覆盖输入项概率。
26. 全局浮窗新增路由栈与面包屑展示（v1 只读）：路径会反映当前编辑层级，为后续 folder/子流程等深层页面导航预留统一机制。
27. 面包屑升级为 v2：支持点击上层路径直接回跳到对应编辑层级（路由栈裁剪）。
28. 顶部头部布局优化：返回/标题/关闭与 header actions 分行展示，actions 行支持横向滚动，减少按钮拥挤与标题压缩。
29. 全局浮窗新增“流程管理”堆叠页：支持 flow 切换、入口节点设置、新增流程、重命名流程、删除当前流程，并复用面包屑回跳。
30. 遮罩点击行为与返回语义统一：若当前处于可返回层级，点弹窗外部先返回上一级；只有根页面才关闭浮窗。
31. Jump/Folder/SubTask 节点新增目标失效提示与“一键修复为当前流程入口”按钮，提升引用修复效率。
32. 动作列表中的 jump 连线支持点击跳转：点击虚线可直接打开目标动作编辑页，便于在长列表中快速定位目标节点。
33. 修复“返回时详情层直接消失”的动效问题：详情层在路由回退时会保留到退场动画结束后再卸载，返回手感与入场动画保持一致。
34. 详情页转场参数优化：进/退场切换为 spring 主导、并同步底层回弹，整体观感更接近“堆叠弹窗”而非硬切换。
35. 浮窗预览图交互增强：节点支持拖拽布局（会话内）、点击节点或连线可直接跳到目标动作编辑；预览区新增无效 jump/folder/subTask 目标修复入口。
36. 修复预览图手势冲突：拖拽节点不再误触发“进入编辑详情”，拖拽结束后会抑制一次 tap，保证拖拽与点击行为可预期。
37. 修复预览图拖拽不生效：点位数据改为实时计算并在手势协程中读取最新状态，拖动节点时可见位置会立即更新。
38. 主页面编辑入口策略收敛：普通编辑入口已下线，任务编辑统一走全局浮窗，主页面仅保留任务库管理与运行入口。
39. 浮窗“动作列表”过滤控制节点：`START/END` 不再展示为可编辑动作项，列表仅显示可编辑节点（如 ACTION/BRANCH/JUMP/FOLDER_REF/SUB_TASK_REF）。
40. `CLICK` 节点编辑新增“屏幕拖动选点”入口：进入全屏拾点层后可直接拖动目标点位并应用到节点；同时保留手动输入，输入语义改为屏幕像素坐标（px）。
41. `CLICK` 参数编辑显示收敛：节点详情中 `x/y` 不再暴露百分比输入框，避免与像素坐标输入形成双语义冲突（内部仍转换为运行时比例值存储）。
42. `CLICK` 拾点交互细化：支持在拾点层任意位置滑动来移动准星，不再要求必须拖拽准星本体。
43. `CLICK` 准星样式改为“空心十字 + 中心点”，减少对底层内容遮挡；位置约束保持“中心点不出屏幕，图标可部分越界”。
44. `CLICK` 拾点提示文案补齐：顶部明确说明“任意位置滑动也可移动准星”，降低误解成本。
45. `CLICK` 拾点底部操作区改为实体卡片背景（非透明）：提升“取消/应用”按钮在复杂底图下的可读性。
46. `CLICK` 节点“屏幕拖动调整点击位置”入口按钮改为整行宽度与更直白文案，降低入口发现成本。
47. 浮窗节点编辑页移除“填充默认值”按钮，减少与手动调参/可视化调参语义冲突。
48. 编辑入口统一：操作面板“任务设置”任务卡片点击改为复用全局任务编辑浮窗（`TaskEditorGlobalOverlay`），与首页任务编辑入口一致，不再走面板内嵌编辑分支。
49. 全局任务编辑浮窗补齐 `CLICK` 可视化编辑：支持 `X(px)/Y(px)` 手动输入 + “屏幕拖动调整点击位置”全屏拾点，交互与操作面板路径保持一致。
50. 全局浮窗 `CLICK` 参数展示收敛：在参数区隐藏比例值 `x/y` 字段，仅保留像素编辑与拾点入口，避免双语义输入混淆。
51. 操作面板到全局编辑器的切换改为“真实堆叠”：从任务设置进入编辑时保留设置浮窗作为下层，编辑浮窗在上层打开，关闭编辑后可回到设置层。
52. 面板来源编辑浮窗新增堆叠视觉参数：上层编辑浮窗在该路径下使用更轻 scrim（便于看到下层）并增加顶部 inset 露出带，强化“上一层仍在”感知。
53. 弹窗样式统一：任务设置与全局编辑浮窗主卡片圆角统一为 18dp。
54. 新增共享对话框组件：`OverlayDialogScaffoldShared`，并用于操作面板设置页三层，统一 `OverlayDialogScaffold` 视觉语义（头部按钮排布、分割线、内容滚动区、底部 actions 区）。
55. 跨窗口堆叠动画联动：从任务设置打开编辑浮窗时，下层设置页会根据编辑浮窗显示态触发退后/回弹动画，补齐“堆叠可见但动画缺失”的体验断层。
56. 跨窗口首层堆叠参数收敛：首层进/退场改为复用同窗堆叠的 `OverlayStackMotion` 参数与 spring 曲线，不再保留独立临时参数，保证“首层 -> 后续层”动画风格一致。
57. 跨窗口层级规则对齐：当编辑浮窗路由深度大于 1（进入详情层）时，任务设置下层自动隐藏，保持全局最多两层可见，避免出现三层混叠。
58. 两层约束时序修复：外部编辑浮窗进入二级页时，下层任务设置层使用即时退场（不保留退出帧），防止过渡瞬间出现第三层；任务设置层 scrim 参数也改为与 `OverlayStackMotion` 同源。
59. 跨窗口监听竞态修复：任务设置侧等待编辑浮窗真实显示后再开始层级监听，避免 `addView` 前轮询提前结束导致“进入详情后下层不变化”。
60. 前景位替换语义对齐：全局编辑浮窗二级页（流程管理/编辑动作）移除额外顶部偏移，进入时直接接管当前前景卡片位置，上一层退到背景位。
61. 面板来源遮挡修复：当从任务设置进入全局编辑后再打开二级页时，下层动作列表会额外向上退让（与缩放同步），避免被二级页整面覆盖，保留可感知堆叠层次。
62. 跨窗口首层对齐：`任务设置 -> 编辑任务` 的下层退让动画补齐为“缩放 + 向上退到背景位”，不再是仅缩放，减少首层与后续层级观感差异。
63. 面板编辑入口改为同窗口路由：从任务设置进入编辑时直接复用 settings overlay 的 `ActionList/NodeEditor` 页面栈，不再打开独立 `TaskEditorGlobalOverlay` 窗口。
64. 同窗口编辑稳定性修复：面板编辑子页面移除内层 `verticalScroll`，改为复用共享 scaffold 的单一滚动容器，避免无限高度约束崩溃。
65. 首页编辑入口收敛：主页面任务卡片点击改为打开“浮窗任务列表（任务设置页）”，首页不再直开 `TaskEditorGlobalOverlay`。
66. 删除全局编辑浮窗实现：`TaskEditorGlobalOverlay` 与服务层相关 API 已移除，编辑交互统一收敛到 settings overlay 单窗口页面栈。

## 3. 已知限制

1. 流程图预览已支持会话内拖拽与连线点击跳转，但布局尚未持久化到任务数据。
2. 参数 schema 已接入默认值补齐与 UI 级校验，但运行时尚未做“插件声明级强校验失败即拒绝执行”。
3. 节点 ID 暂未开放编辑，避免牵扯边和引用联动迁移。
4. 面包屑已支持回跳，但当前未加入“未保存提示后再回跳”的保护流程（后续补充）。
5. 当前编辑页面主要承载在 `TaskControlPanelGlobalOverlay` 内，文件体积和状态职责仍偏大，后续需组件化拆分。
6. 首页不承担直接编辑职责，编辑统一经由“浮窗任务列表 -> 动作列表/动作详情”链路完成。
7. Flow 管理等深层编辑能力仍需继续打磨交互细节（批量操作、引用可视化、布局持久化）。
