# Task List MVP v1（真实任务体验）

更新时间：2026-03-02

## 1. 目标

1. 让用户可以在应用内管理真实任务，而不是仅运行固定 demo。
2. 打通最小闭环：新建任务 -> 编辑动作 -> 保存 -> 任务列表运行 -> 查看最近运行结果。
3. 为后续导入与迁移落地提供统一任务容器（`TaskBundle`）。

## 2. 当前实现

文件：

1. `app/src/main/java/com/ksxkq/cmm_clicker/feature/task/TaskRepository.kt`
2. `app/src/main/java/com/ksxkq/cmm_clicker/ui/MainActivity.kt`
3. `app/src/main/java/com/ksxkq/cmm_clicker/ui/MainTabsRoute.kt`
4. `app/src/main/java/com/ksxkq/cmm_clicker/ui/TaskTabScreen.kt`
5. `app/src/main/java/com/ksxkq/cmm_clicker/ui/ConsoleTabScreen.kt`
6. `app/src/main/java/com/ksxkq/cmm_clicker/ui/MainUiComponents.kt`
7. `app/src/main/java/com/ksxkq/cmm_clicker/accessibility/TaskControlPanelGlobalOverlay.kt`

能力：

1. `LocalFileTaskRepository`：本地 JSON 持久化，支持任务 CRUD 与运行信息更新。
2. 首页任务页（`任务` Tab）：
   - 首页采用 iOS 风格底部导航结构（任务/控制台）
   - 已接入 `Scaffold` 安全区内边距，适配 Android 14 边到边场景
   - 首页只保留一个“打开任务列表”入口卡片
   - 点击入口后打开 Accessibility 浮窗任务列表（任务设置页）
   - 首页不再承载任务 CRUD、搜索筛选、运行与编辑交互
3. 动作详情浮窗：
   - 由 AccessibilityService 承载的全局浮窗（`TYPE_ACCESSIBILITY_OVERLAY`），可在其它 App 页面编辑
   - 统一使用公共弹窗容器：头部标题/菜单 action/底部操作区可按场景配置
   - 浮窗 UI 全量使用 Compose Material3 组件，并通过 `CmmClickerTheme + AppThemeMode` 与首页保持同一主题效果
   - 浮窗改为“全屏弹窗层 + 底部内容面板”，禁止拖动位置，交互更稳定
   - 显示时带背景渐暗动画与底部上滑动画，强化弹窗语义
   - 点击内容外区域可关闭浮窗，交互行为与常规弹窗一致
   - 关闭时带退出动画（背景淡出 + 面板下滑），再执行窗口移除
   - 背景遮罩覆盖状态栏区域，且保持半透明，不使用不透明灰底
   - 默认仅展示动作列表摘要（不铺满参数）
   - 顶部提供“查看预览/隐藏预览”按钮（黑白统一样式），预览支持图形连线与文本连线
   - 动作列表内支持 jump 连接线（同 flow）用于快速识别跳转目标
   - 点击动作后进入“动作编辑浮窗”
4. Jump 设置规则：
   - 跳转语义来自 `NodeKind.JUMP`，不是 `ActionType=jump`
   - 通过 `targetFlowId + targetNodeId` 指向目标节点（支持跨 flow）
5. 浮窗任务列表显示：
   - 更新时间
   - 最近运行时间
   - 最近运行状态
   - 最近运行摘要
   - 搜索（按任务名或 taskId）
   - 分段筛选（全部/最近运行/最近编辑）
   - 删除任务二次确认弹窗
6. 编辑器联动：
   - 进入运行/编辑动作前加载对应 `TaskBundle` 作为当前任务上下文
   - 编辑自动保存（防丢）
   - 手动“保存任务”按钮
7. 控制台联动：
   - 运行按钮语义改为“运行当前任务”。
8. 页面结构：
   - `MainActivity` 仅负责状态与回调分发
   - Tab 页面与 UI 组件拆分到独立文件，便于维护与后续继续 ViewModel 化
9. 全局操作面板（任意 App 界面）：
   - 形态：可拖动矩形工具条（非弹窗卡片）
   - 四按钮：`设置/录制/开始/移除`（仅 icon 展示）
   - `设置`：通过独立 `addView` 打开任务列表浮窗（非面板窗口内切换）
   - 设置页过渡：scrim 渐入 + 列表面板自底部上滑，关闭时反向动画
   - `录制`：全屏录制层采集用户手势，自动落库为 `CLICK/RECORD` 动作
   - `录制`：进入录制会话模式，面板切到 `暂停/停止` 控制；支持连续录制多步手势，停止后弹窗保存为新任务
   - 录制态按钮收敛为 `停止 + 暂停`，不提供独立“关闭录制”按钮；停止即退出录制态
   - 当本次录制步数为 0 时，停止后直接回到普通面板，不弹保存弹窗
   - 录制中的每一步手势在采集完成后会自动回放一次（带轨迹反馈），用于即时确认录制结果
   - 自动回放期间会临时隐藏录制面板和录制浮层，回放结束后恢复录制会话
   - 录制浮层不再覆盖面板视觉区域，保证录制态下暂停/停止可见且可操作
   - 录制浮层改为轻量半透明遮罩（`#12000000`），不再全透明
   - 录制采集支持单指/多指并发，以及“长按后拖动”轨迹（保留时间戳）
   - 自动回放前会临时将录制浮层切为 `NOT_TOUCHABLE`，确保回放手势作用于目标 App，而不是被录制浮层吞掉
   - 操作面板 `普通态 <-> 录制态` 采用“状态尺寸瞬时切换 + 内容淡入淡出”，避免尺寸补间导致的迟滞和形变
   - 面板模式内容切换时长收敛为 `80ms`，减少模式切换拖沓感
   - 录制态进入顺序为“先提高面板层级，再触发内容转场动画”，避免 `remove/addView` 打断动画
   - 录制启动时不再通过 `post` 延迟切换 `panelMode`，避免普通态按钮残留一拍
   - 录制面板右上角显示当前录制时长（暂停不计时）
   - 手势反馈动画时长与真实手势执行时长保持一致（click/swipe/path）
   - 录制动作参数支持 `strokes`（`points/timestampsMs/startDelayMs/durationMs`），运行时兼容旧 `points`
   - 轨迹反馈支持多 stroke 同步渲染（含起始延迟）
   - 单指带时间戳录制在 Android O+ 使用分段 `continueStroke` 回放，支持“长按后拖动”时序
   - 长时录制回放采用动态超时阈值，减少长按场景的超时误报
   - 录制过程中可实时看到触摸轨迹（支持多指），抬手后短暂淡出
   - 录制轨迹与回放轨迹统一橙色系，录制轨迹使用更高透明度以区分来源
   - 回放轨迹进度按 `timestampsMs` 时间轴推进（逐点推进、无段内插值），与 `liteclicker` 轨迹显示一致
   - 长按分段 `continueStroke` 回放模式下，轨迹反馈时长严格使用录制 `duration/timestamps`，不再做按段补偿，避免时间轴被拉长
   - 单指时间戳回放分段改为 `liteclicker FunctionManager` 同款状态机：`startIndex/endIndex/duration/isPause`，并通过 `continueStroke` 链式执行
   - 长按检测采用 `liteclicker` 同款归一化阈值：`maxPauseMove=0.02`、`pauseEntryMove=0.01`、`minPauseDuration=350ms`，仅有效停顿生成 pause 段
   - pause 段执行使用“原地微环 + 1px 收尾”语义，优先保障长按段在实机上的时长稳定性
   - 单指时间戳手势统一走 `timed_continueStroke`，不再切换 `legacy_hold_path`
   - 多指时间戳手势在 Android O+：
     - 同起点（各指 `startDelay` 完全一致）走“统一时间边界 + 多 stroke 同步 `continueStroke`”分段回放
     - 错峰起点（例如先按第一指再按第二指）自动回退单次 multi-stroke 分发，优先避免系统取消整段手势
   - 录制采样改为“尽量全量保留 move 点 + 原始时间戳”（仅去重同事件同坐标点），不再做录制端抽样压缩，避免长按停顿时长被压短
   - 录制落库取消二次 downsample，回放停顿判定基于原始 `points/timestamps`
   - 回放调试新增 `expected vs actual` 时长日志，便于识别机型级分发偏慢
   - 新增录制与回放诊断日志：输出 `track/stroke` 时序摘要（points、duration、tsLast、startDelay）和分段明细（pause/move）
   - 分发失败日志细化：失败 detail 含 `segment/interval index + start/end/duration + active + reason`，可直接定位双指回放失败区间
   - 新增统一 `traceId` 的“动画-执行对时”日志：同一次回放可同时看到反馈层 `request/add/25-100%/detach` 与执行层 `dispatch start/end/cancel`，用于拆解速度不一致来源
   - 对时基准已收敛：反馈进度以 `onAttachedToWindow` 为起点，执行链路在回放前增加一帧同步延迟，降低动画与执行起步错位
   - 录制保存弹窗动画已修复：采用 `openToken + 首帧后进入` 触发入场，关闭时按退出动画状态收敛后再移除 settings overlay，确保入场/退场动画都可见
   - 保存弹窗关闭态新增 `animatingOut` 过渡保持，避免因 overlay 内容早退清空导致退出动画被截断
   - 开始任务确认弹窗动画机制与保存弹窗对齐：同样使用 `openToken + 首帧后进入 + animatingOut`，并在退出动画完成（或被反向中断）后再判定是否移除 overlay，避免 `addView/removeView` 抢动画
   - 设置页与中心确认弹窗的 scrim 统一为同一套透明度与淡入淡出时长参数，避免“任务设置”和“确认弹窗”背景观感不一致
   - settings overlay 的 `removeView` 时机进一步收敛为“内容退出 + scrim 渐隐完成”后执行，消除“背景层突然消失”体感
   - 浮层中心弹窗样式统一由 `OverlayDialogCardScaffold` 管理（圆角/边框/间距/标题与说明文案结构统一）
   - 录制保存弹窗仅承担“命名 + 保存/丢弃”确认，不提供动作编辑能力
   - 动作编辑统一在任务编辑页进行（动作列表/动作详情），避免录制保存与动作编辑职责混杂
   - 错峰多指回退路径的长按建模已改为“按时间加权路径长度”，按移动段参考速度分配长按路径占比，提升“先长按再滑动”的时长还原度
   - 当前回放策略优先“时长正确性”：pause 段允许微环抖动以提高不同 ROM 下的长按一致性
   - 面板高度按内容包裹，不再使用最小高度硬编码
   - 面板重挂场景下通过保留组合树与精简 re-stack 步骤降低闪烁
   - `开始`：点击后先弹确认弹窗，确认后执行最近启动任务（无记录时回退到当前选择任务）
   - `移除`：退出动画后移除面板
   - 显示动画：渐隐 + 轻微缩放 + 轻微位移
   - 关闭动画：反向渐隐 + 轻微缩放 + 轻微位移
   - 窗口尺寸：工具条模式严格按内容 `wrap_content` 渲染（不铺满）
   - 视觉约束：工具条模式无额外半透明灰底层
   - 拖拽行为：支持在屏幕内拖拽定位，返回工具条后保持当前位置
   - 设置弹窗关闭后直接回到现有工具条状态，不重新播放入场动画
   - 设置页任务卡片点击后与首页保持同一编辑入口（浮窗任务列表 -> 同窗口编辑栈），形成“任务设置 -> 编辑任务”的真实堆叠
   - 面板来源的编辑浮窗启用“堆叠视觉模式”：编辑层使用更轻背景遮罩并增加顶部露出 inset，确保可见下层任务设置卡片
   - 面板来源跨窗口联动补齐：编辑浮窗打开/关闭时，下层任务设置会同步执行“退后/回弹”动画，避免“只看到叠放、看不到压栈动画”
   - 跨窗口堆叠参数与同窗堆叠统一：首层进入编辑时不再使用独立临时参数，改为复用 `OverlayStackMotion` 同套 scrim/位移/弹簧节奏
   - 跨窗口可见层级限制恢复为 2：编辑浮窗进入二级页时，下层任务设置 sheet 自动隐藏，避免出现“任务设置 + 编辑一级 + 编辑二级”三层同时可见
   - 两层限制的过渡帧修复：进入编辑二级页时，下层任务设置 sheet 使用即时退场（不保留 exit 帧），避免短暂出现第三层
   - 首层背景遮罩一致性修复：任务设置层在非中心弹窗场景下改为复用 `OverlayStackMotion.SHEET_SCRIM_ALPHA`，与编辑堆叠层观感保持一致
   - 跨窗口监听竞态修复：任务设置侧会先等待编辑浮窗真实显示再启用 routeDepth 联动，避免“编辑详情已打开但下层仍停在原状态”
   - 编辑器二级页遮挡修复：面板来源路径下，进入动作详情时“编辑任务”层会向上退让而非原地被遮盖，堆叠层级持续可见
   - 首层退让一致性修复：`任务设置 -> 编辑任务` 时，下层任务设置也会向上退到背景位（而非仅缩放），与后续层级过渡语义对齐
   - 最终入口收敛：面板侧“任务设置 -> 编辑任务”已改为同窗口路由（settings overlay 内继续入栈），不再通过独立全局编辑窗口跨层叠加
   - 同窗口编辑稳定性修复：移除编辑子页面内层滚动容器，统一由共享 scaffold 提供滚动，避免无限高度约束导致崩溃
   - 首页入口同步收敛：任务页点击改为“打开浮窗任务列表（任务设置页）”，不再从首页直开全局编辑器
   - 删除全局编辑浮窗实现：`TaskEditorGlobalOverlay` 与对应服务接口已下线，任务编辑统一由 settings overlay 单窗口路由承载
   - 任务设置与编辑任务的主弹窗圆角统一（18dp）
   - `任务设置/动作列表/编辑动作` 三层页面统一复用共享 `OverlayDialogScaffold` 样式壳（标题区、按钮区、内容区、边框圆角），不再使用面板侧单独 header/card 实现
   - 面板设置页职责升级为“任务库 + 同窗口编辑入口”，动作列表/动作详情在 settings overlay 内继续入栈
   - `CLICK` 节点支持“像素输入 + 屏幕拖动选点”，首页与面板最终都走同一套浮窗编辑能力
   - 设置页堆叠转场按“整页层”执行：标题栏与内容区一起参与进退场动画
   - 堆叠视觉强化：动作层/详情层使用不同 inset 卡片层级，底层同步缩放与位移，增强“页面压栈”感知
   - 设置页堆叠动效参数与首页编辑器统一为 spring 上滑/回落节奏，避免两处动效观感不一致
   - 设置页不再使用“固定外壳 + 内部切换”；改为每一层页面独立卡片参与堆叠，层级轮廓会随路由变化
   - 三层堆叠时底层继续退让：第 3 层出现会触发第 1/2 层额外缩放、位移和变暗，不再只看到“新层覆盖旧层”
   - 堆叠语义对齐 iOS：新出现的上层页面保持主体尺寸，主要由下层页面执行缩小/退让动画
   - 首页编辑浮窗与设置浮窗共用同一套堆叠参数（`OverlayStackMotion`），两条编辑入口的动画观感保持一致
   - 三层堆叠的顶部露出高度已回调到统一规范，避免第 3 层露出过多
   - 三层堆叠强化“第二层可见带”：通过增大 `SECOND/THIRD` 顶部间距差与下层向下退让，避免第三层完全遮挡第二层
   - 设置浮窗可见堆叠深度限制为 2：第 3 级页面出现时隐藏最底层，仅显示“上一层 + 当前层”，避免多层叠加导致信息噪声
   - 视觉细化：`2->3` 时中间层不再缩放（保持主体高度感），`1->2` 通过底层退让增强露出，不依赖压缩上层高度
   - `1->2` 堆叠保证首层顶部露出带：采用轻缩放 + 轻微上移 + 第二层固定顶部 inset，避免“切页后首层完全不可见”
   - `2->3` 迭代规则：新页面进入时占据上一层位置，上一层后移到背景位置；不再出现“新层直接在更低位置覆盖”的非迭代感
   - 底部露出收敛：降低堆叠缩放幅度，减少卡片底部缩窄和空隙
   - 最新参数：为增强层级感，后层缩放恢复到 `0.965`；`2->3` 时背景层位置回到 `topInset=0`，新层仍保持前景 `topInset=18`
   - 露出带增强：前景 `topInset` 提升到 `24dp`，下层顶部露出更明显
   - `2->3` 位移平滑：中间层从前景位退到背景位时使用 spring 动画，避免“往上顶一下”的突兀跳变
   - 堆叠常量命名已收敛到“两层可见”语义，避免继续出现“第三层参数”误导
   - inset 动画加了非负保护，避免深层打开时出现 `Padding must be non-negative` 崩溃
   - 设置弹窗关闭时，操作面板采用常驻节点显隐，避免返回工具条瞬间闪烁
   - 设置弹窗内部空白区点击会被内容容器消费，不会冒泡到遮罩导致误返回/误关闭
   - 设置弹窗遮罩改为独立底层点击区域，确保任务列表与编辑页可正常滚动
   - 任务设置页为 `TaskLibraryPanel` 提供独立纵向滚动承载层，任务数量较多时仍可滑动浏览
   - 动作列表与动作详情顶部提供返回按钮，可逐级返回到任务列表
   - 控制台页提供“打开操作面板/关闭操作面板”入口，用于启动全局浮窗

## 3. 当前限制

1. 持久化当前为文件 JSON，尚未升级到 Room（后续可平滑替换）。
2. 任务列表当前尚未支持批量操作（批量删除/批量运行）。
3. 任务模板目前仅用于研发阶段，正式上线可关闭默认模板创建。

5. 目前仍是 Activity 内状态持有，下一步再下沉到独立 ViewModel/RouteState。
6. 录制会话当前支持“连续采集 + 停止后保存新任务”，后续再补“录制结果回放校对、单步删除/重排”。
7. 多指录制单次手势受系统 `dispatchGesture` stroke 数限制，当前实现安全上限为 18 条 stroke。
